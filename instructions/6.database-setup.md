# 6. 数据库初始化与迁移

1. 在 `backend/.env` 中填写数据库连接信息（可参考 `.env.example`）
2. 启动本地 Postgres（Docker 或本地服务均可）
3. 使用 Alembic 生成首个迁移
   ```bash
   cd backend
   alembic revision --autogenerate -m "init"
   ```
4. 检查 `migrations/versions/*` 文件，确认模型与表结构一致
5. 应用迁移
   ```bash
   alembic upgrade head
   ```
6. 验证表结构
   - 使用 `psql`：`\dt` 查看表，`\d tablename` 检查字段
   - 或使用 GUI（TablePlus、DBeaver 等）
7. 如果模型更新：重复“autogenerate → 检查 → upgrade”流程，确保多人协作时迁移记录清晰
