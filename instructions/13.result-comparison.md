# 13. 结果比对与任务详情（当前状态 + 规划）

> 目前仓库尚未实现“结果比对/任务详情”接口，只有 `ReplayResult` 模型定义。以下内容说明现状和下一步需要做的事情。

---

## 1. 数据模型（已存在）

`app/models.py` 已定义 `ReplayResult`，字段如下：
```python
class ReplayResult(Base):
    __tablename__ = "replay_results"
    id = Column(Integer, primary_key=True, index=True)
    task_id = Column(Integer, ForeignKey("replay_tasks.id"), index=True, nullable=False)
    request_index = Column(Integer)
    original_request = Column(JSON)
    original_response = Column(JSON)
    replayed_request = Column(JSON)
    replayed_response = Column(JSON)
    status = Column(String(50))  # success/failed/diff
    diff_details = Column(JSON)
    response_time = Column(Integer)
    created_at = Column(DateTime(timezone=True), server_default=func.now())
```

目前没有生成 `ReplayResult` 的逻辑，也没有对 `diff_details` 做任何处理。

---

## 2. 现有 API

- `app/api/tasks.py` 只实现了任务列表和创建
- 没有任务详情接口 (`GET /tasks/{id}`) 和结果列表接口 (`GET /tasks/{id}/results`)
- 前后端也没有结果比对、进度展示的实现

---

## 3. 下一步实现建议

1. **生成结果**：在 Celery 任务 `run_replay_task` 中读取流量文件，逐条请求下游服务，保存 `ReplayResult`
2. **比对逻辑**：可在 `app/services/result_comparator.py` 编写对比函数，支持忽略字段、输出差异
3. **API**：
   - `GET /api/v1/tasks/{id}`：返回任务基本信息（环境、流量文件、统计数据）
   - `GET /api/v1/tasks/{id}/results`：分页返回 `ReplayResult`，支持 `status` 过滤
4. **前端**：
   - 任务详情页：展示任务进度、统计、关联环境/文件
   - 结果列表：表格 + diff 详情弹窗
5. **扩展**：导出报告、配置 diff 规则、支持忽略字段列表等
