# 8. 前端功能联调步骤

1. **配置环境变量**
   - 在 `frontend/.env` 中设置 `VITE_API_BASE_URL=http://localhost:8000`
2. **任务列表页面**
   - 读取 `GET /api/v1/tasks/`
   - 支持 `page/page_size/status/search`
   - 表格展示 `id/name/status/progress`，提供“查看详情”入口
3. **任务详情页面**（可选）
   - 调用 `GET /api/v1/tasks/{id}`、`GET /api/v1/tasks/{id}/results`
   - 展示基础信息 + 进度条 + 结果列表
4. **流量管理页面**
   - 上传：`POST /api/v1/traffic/upload`（FormData）
   - 列表：`GET /api/v1/traffic/`，支持搜索
5. **环境管理页面**
   - 新建环境：表单提交到 `POST /api/v1/environments/`
   - 列表展示现有环境，可作为创建任务时的下拉选项
6. **全局状态与 UI**
   - 可使用 Pinia 或组合式 API 管理任务列表/环境列表
   - Element Plus / Ant Design Vue 等 UI 库可按需引入
7. **联调注意事项**
   - 确保 CORS 配置允许前端地址
   - 通过浏览器开发者工具检查请求/响应与错误日志
   - 遇到跨域或 401 问题，先确认后端中间件与 Token 逻辑
