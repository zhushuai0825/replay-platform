# 5. 下一步 & 实施顺序

1. **依次执行 1~4 号文档**，确保环境、后端、前端、Docker 逐步完成
2. 在每个阶段结束后，运行以下自检：
   - 后端：`pytest`（如已配置测试）或 `uvicorn` 本地启动
   - 前端：`npm run dev` 并访问 `http://localhost:5173`
   - Docker：`docker compose up -d` 后访问前后端、检查 `docker compose ps`
3. 将实际操作记录到 `learning-notes/daily/日期.md`，便于回顾
4. 遇到问题优先查看日志：
   - 后端：`backend/logs/app.log` 或 `docker compose logs backend`
   - 前端：浏览器控制台 + `docker compose logs frontend`
   - 数据库：`docker compose logs postgres`
5. 按需求扩展功能：
   - Web UI：任务详情页、结果 diff 展示
   - 后端：结果比对逻辑、鉴权体系、指标采集
   - 运维：CI/CD、Prometheus + Grafana 监控
