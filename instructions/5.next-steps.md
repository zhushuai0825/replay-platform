# 5. 下一步 & 实施顺序

1. **依次执行 1~4 号文档**，建议顺序：环境准备 → 后端脚手架 → 前端脚手架 → Docker 集成
   - 每完成一个步骤就进行本地验证，避免问题积累
2. 每个阶段结束后的自检：
   - 后端：`uvicorn app.main:app --reload` 或 `pytest`
   - 前端：`npm run dev`，访问 `http://localhost:5173`
   - Docker：`docker compose up -d`，访问 http://localhost:8000/docs 与 http://localhost:3000
3. 将操作过程记录到 `learning-notes/daily/日期.md`，包括命令、输出结果、问题与解决办法
4. 遇到问题优先查看日志：
   - 后端：`backend/logs/app.log`、`docker compose logs backend`
   - 前端：浏览器控制台、`docker compose logs frontend`
   - 数据库：`docker compose logs postgres`
5. 按需扩展功能：
   - Web UI：任务详情页、结果 diff 展示
   - 后端：结果比对逻辑、鉴权体系、指标采集
   - 运维：CI/CD、Prometheus + Grafana 监控
